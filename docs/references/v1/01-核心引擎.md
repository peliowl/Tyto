# 核心引擎

本文档描述 Tyto UI 的底层引擎模块，包括主题引擎 `ThemeEngine` 和设计令牌数据模型 `DesignTokenSet`。

---

## 1. DesignTokenSet — 设计令牌数据模型

`DesignTokenSet` 是一个 `dataclass`，表示单个主题的完整设计令牌集合。每个主题（如 light、dark）对应一个实例。

### 1.1 数据结构

```python
@dataclass
class DesignTokenSet:
    name: str                                      # 主题标识符，如 "light"、"dark"
    colors: dict[str, str]                         # 语义化颜色令牌 → CSS 颜色字符串
    spacing: dict[str, int]                        # 间距令牌 → 像素值
    radius: dict[str, int]                         # 圆角令牌 → 像素值
    font_sizes: dict[str, int]                     # 字号令牌 → 像素值
    shadows: dict[str, str]                        # 阴影令牌 → CSS box-shadow 字符串
    component_sizes: dict[str, dict[str, int]]     # 组件尺寸变体（height, padding_h, font_size, icon_size）
    switch_sizes: dict[str, dict[str, int]]        # Switch 尺寸变体（width, height, thumb）
    spin_sizes: dict[str, dict[str, int]]          # Spin 尺寸变体
    slider: dict[str, int]                         # Slider 令牌（track_height, thumb_size, thumb_border_size）
    layout: dict[str, Any]                         # Layout 令牌（header_height, footer_height, sider_width 等）
    card: dict[str, int]                           # Card 令牌（padding_small, padding_medium, padding_large）
    menu: dict[str, int]                           # Menu 令牌（indent, item_height, item_padding_h, collapsed_width）
```

### 1.2 必需令牌键

每个类别都有一组必需键，加载时会进行严格校验：

| 类别 | 必需键 |
|------|--------|
| `colors` | `primary`, `primary_hover`, `primary_pressed`, `success`, `warning`, `error`, `info`, `info_hover`, `info_pressed`, `bg_default`, `bg_elevated`, `text_primary`, `text_secondary`, `text_disabled`, `border`, `border_focus`, `white`, `mask` |
| `spacing` | `small`, `medium`, `large`, `xlarge` |
| `radius` | `small`, `medium`, `large` |
| `font_sizes` | `small`, `medium`, `large`, `xlarge` |
| `shadows` | `small`, `medium`, `large` |

`component_sizes` 和 `switch_sizes` 为可选类别，不参与必需键校验。

### 1.3 组件尺寸令牌（component_sizes）

用于统一管理各尺寸变体下的组件维度，键名对应尺寸名称（如 `tiny`、`small`、`medium`、`large`）：

```json
{
  "component_sizes": {
    "tiny": { "height": 22, "padding_h": 6, "font_size": 12, "icon_size": 14 },
    "small": { "height": 28, "padding_h": 10, "font_size": 13, "icon_size": 16 },
    "medium": { "height": 34, "padding_h": 14, "font_size": 14, "icon_size": 18 },
    "large": { "height": 40, "padding_h": 18, "font_size": 15, "icon_size": 20 }
  }
}
```

### 1.4 Switch 尺寸令牌（switch_sizes）

用于统一管理 TSwitch 各尺寸变体下的轨道和滑块维度：

```json
{
  "switch_sizes": {
    "small": { "width": 32, "height": 18, "thumb": 14 },
    "medium": { "width": 40, "height": 22, "thumb": 18 },
    "large": { "width": 50, "height": 26, "thumb": 22 }
  }
}
```

### 1.5 方法

| 方法 | 返回值 | 说明 |
|------|--------|------|
| `to_dict()` | `dict[str, Any]` | 将令牌集序列化为 JSON 兼容的字典（包含 component_sizes 和 switch_sizes） |

### 1.6 加载函数

```python
def load_tokens_from_file(path: str | Path) -> DesignTokenSet
```

从 JSON 文件加载并校验设计令牌。校验失败时抛出 `TokenFileError`，文件不存在时抛出 `FileNotFoundError`。

### 1.7 异常类

```python
class TokenFileError(Exception):
    message: str    # 错误描述
    location: str   # 定位信息（如缺失的键名）
```

## 2. ThemeEngine — 主题引擎

`ThemeEngine` 是一个单例 QObject，负责管理设计令牌、渲染 QSS 模板、以及在主题切换时通知所有组件更新样式。

### 2.1 单例模式

```python
engine = ThemeEngine.instance()   # 获取单例
ThemeEngine.reset()               # 重置单例（仅用于测试）
```

### 2.2 信号

| 信号 | 参数 | 说明 |
|------|------|------|
| `theme_changed` | `str` | 主题切换后发射，携带新主题名称 |

### 2.3 公开方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `load_tokens` | `(tokens_dir: str \| Path \| None) -> None` | 从目录加载所有主题 JSON 文件，并初始化 Jinja2 模板环境。默认使用内置 `styles/tokens/` 目录 |
| `switch_theme` | `(theme_name: str) -> None` | 切换到指定主题，重新渲染全局 QSS 并发射 `theme_changed` 信号 |
| `get_token` | `(category: str, key: str) -> str \| int` | 获取当前主题中指定类别和键的令牌值 |
| `current_theme` | `() -> str` | 返回当前活跃主题名称，无活跃主题时返回空字符串 |
| `current_tokens` | `() -> DesignTokenSet \| None` | 返回当前主题的令牌集实例 |
| `render_qss` | `(template_name: str, **extra_context) -> str` | 使用当前令牌渲染指定 Jinja2 QSS 模板 |

### 2.4 渲染上下文

`render_qss` 方法向 Jinja2 模板注入以下上下文变量：

| 变量 | 类型 | 说明 |
|------|------|------|
| `colors` | `dict[str, str]` | 颜色令牌字典 |
| `spacing` | `dict[str, int]` | 间距令牌字典 |
| `radius` | `dict[str, int]` | 圆角令牌字典 |
| `font_sizes` | `dict[str, int]` | 字号令牌字典 |
| `shadows` | `dict[str, str]` | 阴影令牌字典 |
| `component_sizes` | `dict[str, dict[str, int]]` | 组件尺寸变体字典 |
| `switch_sizes` | `dict[str, dict[str, int]]` | Switch 尺寸变体字典 |
| `spin_sizes` | `dict[str, dict[str, int]]` | Spin 尺寸变体字典 |
| `slider` | `dict[str, int]` | Slider 令牌字典 |
| `layout` | `dict[str, Any]` | Layout 令牌字典 |
| `card` | `dict[str, int]` | Card 令牌字典 |
| `menu` | `dict[str, int]` | Menu 令牌字典 |
| `theme` | `str` | 当前主题名称 |

### 2.5 工作流程

```
load_tokens()          加载 JSON → 校验 → 构建 DesignTokenSet 字典
       ↓                        初始化 Jinja2 FileSystemLoader
switch_theme("light")  选择主题 → 渲染所有 QSS 模板 → 设置全局样式表
       ↓                        发射 theme_changed 信号
BaseWidget._on_theme_changed()  各组件接收信号 → 调用 apply_theme()
```

### 2.6 使用示例

```python
from tyto_ui_lib import ThemeEngine

engine = ThemeEngine.instance()
engine.load_tokens()                              # 加载内置令牌
engine.switch_theme("light")                      # 激活 Light 主题

primary = engine.get_token("colors", "primary")   # "#18a058"
spacing = engine.get_token("spacing", "medium")   # 8

qss = engine.render_qss("button.qss.j2")         # 渲染按钮样式
```

## 3. EasingEngine — 缓动动画引擎

`EasingEngine` 提供标准缓动函数（cubic、quadratic）和自定义 cubic-bezier 曲线支持。所有函数接受归一化时间 t ∈ [0.0, 1.0]，返回归一化进度 ∈ [0.0, 1.0]。超出范围的 t 值自动 clamp。

### 3.1 静态方法 — Cubic 缓动

| 方法 | 公式 | 说明 |
|------|------|------|
| `ease_in_cubic(t)` | t³ | 加速进入 |
| `ease_out_cubic(t)` | 1 - (1-t)³ | 减速退出 |
| `ease_in_out_cubic(t)` | 分段 | 平滑加速减速 |

### 3.2 静态方法 — Quadratic 缓动

| 方法 | 公式 | 说明 |
|------|------|------|
| `ease_in_quad(t)` | t² | 加速进入 |
| `ease_out_quad(t)` | 1 - (1-t)² | 减速退出 |
| `ease_in_out_quad(t)` | 分段 | 平滑加速减速 |

### 3.3 自定义贝塞尔曲线

```python
@staticmethod
def custom_bezier(p1x: float, p1y: float, p2x: float, p2y: float) -> Callable[[float], float]
```

创建自定义 cubic-bezier 缓动函数，等价于 CSS `cubic-bezier(p1x, p1y, p2x, p2y)`。使用 Newton-Raphson 迭代 + 二分法回退求解。

### 3.4 使用示例

```python
from tyto_ui_lib import EasingEngine

progress = EasingEngine.ease_in_out_cubic(0.5)

ease = EasingEngine.custom_bezier(0.25, 0.1, 0.25, 1.0)
progress = ease(0.5)
```

## 4. EventBus — 全局事件总线

`EventBus` 是一个单例发布/订阅消息总线，支持组件间解耦通信，无需直接的父子关系或 Qt 信号连接。

### 4.1 单例模式

```python
bus = EventBus.instance()   # 获取单例
EventBus.reset()            # 重置单例（仅用于测试）
```

### 4.2 公开方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `on` | `(event_name: str, callback: Any) -> None` | 订阅事件 |
| `off` | `(event_name: str, callback: Any) -> None` | 取消订阅 |
| `once` | `(event_name: str, callback: Any) -> None` | 订阅事件（仅触发一次后自动取消） |
| `emit` | `(event_name: str, *args, **kwargs) -> None` | 发布事件，按订阅顺序调用所有回调 |
| `clear` | `(event_name: str) -> None` | 移除指定事件的所有订阅 |
| `clear_all` | `() -> None` | 移除所有事件的所有订阅 |

### 4.3 错误处理

回调执行时如果抛出异常，异常会被记录到日志，剩余回调继续执行，不会中断事件传播。

### 4.4 使用示例

```python
from tyto_ui_lib import EventBus

bus = EventBus.instance()
bus.on("user:login", lambda name: print(f"Welcome, {name}"))
bus.emit("user:login", "Alice")  # 输出: Welcome, Alice

bus.once("app:ready", lambda: print("App is ready"))
bus.emit("app:ready")   # 输出: App is ready
bus.emit("app:ready")   # 无输出（已自动取消订阅）
```
