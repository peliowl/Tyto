# 样式系统

本文档描述 Tyto UI 的 Design Token + Jinja2 + QSS 动态样式架构。

---

## 1. 架构概述

```
Token JSON 文件 (light.json / dark.json)
       ↓ load_tokens_from_file()
DesignTokenSet (内存数据模型)
       ↓ ThemeEngine.render_qss()
Jinja2 QSS 模板 (*.qss.j2)
       ↓ 模板渲染
QSS 字符串
       ↓ QApplication.setStyleSheet() / widget.setStyleSheet()
Qt 样式引擎应用样式
```

核心原则：组件代码中严禁硬编码任何像素值、颜色值或字体大小，所有视觉属性必须通过 Design Token 获取。

---

## 2. Design Token 规范

### 2.1 Token 文件格式

Token 文件为 JSON 格式，存放在 `styles/tokens/` 目录下，每个文件代表一个主题。

### 2.3 间距令牌（spacing）

| 令牌键 | 值 (px) | 典型用途 |
|--------|---------|----------|
| `small` | 4 | 紧凑内边距、图标间距 |
| `medium` | 8 | 标准内边距 |
| `large` | 16 | 宽松内边距、区块间距 |
| `xlarge` | 24 | 大区块间距 |

### 2.4 圆角令牌（radius）

| 令牌键 | 值 (px) | 典型用途 |
|--------|---------|----------|
| `small` | 2 | Checkbox 指示器、Tag |
| `medium` | 4 | Button、Input、Message |
| `large` | 8 | Modal 对话框 |

### 2.5 字号令牌（font_sizes）

| 令牌键 | 值 (px) | 典型用途 |
|--------|---------|----------|
| `small` | 12 | Tag (small)、描述文本 |
| `medium` | 14 | 默认正文、按钮文本 |
| `large` | 16 | Modal 标题、Tag (large) |
| `xlarge` | 20 | 页面标题 |

### 2.6 阴影令牌（shadows）

| 令牌键 | Light 主题值 | Dark 主题值 |
|--------|-------------|-------------|
| `small` | `0 2px 8px rgba(0,0,0,0.08)` | `0 2px 8px rgba(0,0,0,0.24)` |

## 3. QSS 模板系统

### 3.1 模板文件列表

| 模板文件 | 对应组件 |
|----------|----------|
| `button.qss.j2` | TButton |
| `checkbox.qss.j2` | TCheckbox |
| `radio.qss.j2` | TRadio / TRadioGroup |
| `input.qss.j2` | TInput |
| `switch.qss.j2` | TSwitch |
| `tag.qss.j2` | TTag |
| `searchbar.qss.j2` | TSearchBar |
| `breadcrumb.qss.j2` | TBreadcrumb |
| `inputgroup.qss.j2` | TInputGroup |
| `message.qss.j2` | TMessage |
| `modal.qss.j2` | TModal |

### 3.2 模板变量

所有模板共享以下 Jinja2 上下文变量：

| 变量 | 类型 | 说明 |
|------|------|------|
| `colors` | `dict[str, str]` | 颜色令牌字典 |
| `spacing` | `dict[str, int]` | 间距令牌字典 |
| `radius` | `dict[str, int]` | 圆角令牌字典 |
| `font_sizes` | `dict[str, int]` | 字号令牌字典 |
| `shadows` | `dict[str, str]` | 阴影令牌字典 |
| `theme` | `str` | 当前主题名称 |

### 3.3 模板语法示例

```jinja2
TButton {
    background-color: {{ colors.bg_default }};
    color: {{ colors.text_primary }};
    border: 1px solid {{ colors.border }};
    border-radius: {{ radius.medium }}px;
    padding: {{ spacing.small }}px {{ spacing.large }}px;
    font-size: {{ font_sizes.medium }}px;
}

TButton[buttonType="primary"] {
    background-color: {{ colors.primary }};
    color: {{ colors.white }};
}
```

### 3.4 动态属性选择器

组件通过 `setProperty()` 设置动态属性，QSS 通过属性选择器匹配：

| 属性 | 组件 | 可选值 |
|------|------|--------|
| `buttonType` | TButton | `"primary"`, `"default"`, `"dashed"`, `"text"` |
| `checkState` | TCheckbox | `"unchecked"`, `"checked"`, `"indeterminate"` |
| `checked` | TRadio, TSwitch | `true`, `false` |
| `focused` | 使用 FocusGlowMixin 的组件 | `true`, `false` |
| `pressed` | 使用 ClickRippleMixin 的组件 | `true`, `false` |
| `messageType` | TMessage | `"info"`, `"success"`, `"warning"`, `"error"` |
| `tagType` | TTag | `"default"`, `"primary"`, `"success"`, `"warning"`, `"error"` |
| `tagSize` | TTag | `"small"`, `"medium"`, `"large"` |
| `groupPosition` | TInputGroup 子组件 | `"solo"`, `"first"`, `"middle"`, `"last"` |
| `clickable` | TBreadcrumb 子标签 | `"true"` |
| `current` | TBreadcrumb 子标签 | `"true"` |
| `separator` | TBreadcrumb 分隔符 | `"true"` |

## 4. 主题切换流程

```
用户触发 → engine.switch_theme("dark")
  1. 验证主题名称存在
  2. 更新 _current_theme
  3. 渲染所有 *.qss.j2 模板并拼接
  4. 调用 QApplication.setStyleSheet(全局 QSS)
  5. 发射 theme_changed("dark") 信号
  6. 各 BaseWidget 子类接收信号 → apply_theme()
  7. 各组件调用 engine.render_qss() 获取组件级 QSS
  8. 调用 self.setStyleSheet(组件 QSS)
```

全局 QSS 和组件级 QSS 双重应用，确保样式完整覆盖且无闪烁。

---

## 5. 自定义主题

创建新主题只需在 `styles/tokens/` 目录下添加一个符合规范的 JSON 文件：

```json
{
  "name": "my-theme",
  "colors": { ... },
  "spacing": { ... },
  "radius": { ... },
  "font_sizes": { ... },
  "shadows": { ... }
}
```

调用 `engine.load_tokens()` 后即可通过 `engine.switch_theme("my-theme")` 激活。
